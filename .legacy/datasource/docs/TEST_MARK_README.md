# Mark 批量请求压测脚本使用指南

## 📋 功能说明

这个 Python 脚本用于测试 datasource 的 mark 批量接口性能，具有以下特点：

- ✅ 10个不同的请求名称（模拟真实业务场景）
- ✅ 每秒发送10次批量请求（可配置）
- ✅ 每次批量请求包含10个 mark（对应10个不同的请求名称）
- ✅ 模拟真实响应时间（10-500ms）
- ✅ 10% 的失败率模拟
- ✅ 实时统计和监控（每5秒）
- ✅ **每10秒自动查询并显示最新 Metrics（列表格式）**
- ✅ 自动查询最终的 Metrics 数据

## 🔧 环境准备

### 1. 安装 Python 依赖

```bash
pip3 install requests
```

### 2. 确保服务正在运行

```bash
# 检查服务是否启动
curl http://localhost:8080/health
```

## 🚀 使用方法

### 基础用法（使用默认参数）

```bash
# 使用默认配置运行60秒
python3 test_mark_batch.py
```

**默认参数：**
- 服务地址: `localhost:8080`
- 压测ID: `test_stress_001`
- QPS: `10`（每秒10次批量请求）
- 时长: `60秒`

### 自定义参数

```bash
# 完整参数示例
python3 test_mark_batch.py \
    --host localhost \
    --port 8080 \
    --stress-id my_stress_test_001 \
    --qps 20 \
    --duration 120
```

### 参数说明

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `--host` | 服务器地址 | `localhost` | `--host 192.168.1.100` |
| `--port` | 服务器端口 | `8080` | `--port 8888` |
| `--stress-id` | 压测ID（唯一标识） | `test_stress_001` | `--stress-id prod_test_001` |
| `--qps` | 每秒请求数 | `10` | `--qps 50` |
| `--duration` | 运行时长（秒） | `60` | `--duration 300` |

## 📊 测试场景

### 场景1：低压力测试（默认）


## 🔍 监控和验证

### 1. 查看实时 Metrics

在测试运行期间，可以使用另一个终端查询实时数据：

```bash
curl -X POST http://localhost:8080/apis/v1/mark/list \
  -H "Content-Type: application/json" \
  -d '{"query": "{\"stress_id\": \"test_stress_001\"}"}'
```

### 2. 查看 TSDB 中的数据

等待聚合器运行（默认15秒一次），然后查询 TSDB：

```bash
curl -X POST http://localhost:8080/apis/v1/metrics/query \
  -H "Content-Type: application/json" \
  -d '{
    "metric_name": "total_num",
    "labels": ["stress_id", "test_stress_001"],
    "start_time": 0,
    "end_time": 0,
    "limit": 100
  }'
```

### 3. 使用 PromQL 查询

```bash
curl -X POST http://localhost:8080/apis/v1/metrics/query \
  -H "Content-Type: application/json" \
  -d '{
    "promql": "total_num{stress_id=\"test_stress_001\"}",
    "start_time": 0,
    "end_time": 0
  }'
```

## 💡 提示和建议

### 性能测试建议

1. **逐步增加压力**：从低 QPS 开始，逐步增加到目标值
2. **观察资源使用**：使用 `top` 或 `htop` 监控 CPU 和内存
3. **检查日志**：观察服务日志是否有异常
4. **验证数据准确性**：对比发送的数据和查询到的 Metrics

### 常见问题

**Q: 脚本提示连接失败？**
```
A: 检查服务是否启动：curl http://localhost:8080/health
```

**Q: QPS 达不到目标值？**
```
A: 可能是服务器性能瓶颈，检查：
   1. 服务器 CPU/内存使用率
   2. 网络延迟
   3. 数据库性能
```

**Q: 如何停止测试？**
```
A: 按 Ctrl+C 优雅停止，脚本会打印最终统计
```

**Q: 如何清理测试数据？**
```bash
# 设置 mark 过期
curl -X POST http://localhost:8080/apis/v1/mark/set_expired \
  -H "Content-Type: application/json" \
  -d '{"stress_id": "test_stress_001"}'
```

## 📊 Metrics 列表说明

脚本会以表格形式显示每个请求的关键指标：

| 列名 | 说明 | 单位 |
|------|------|------|
| **请求名称** | API 请求的名称 | - |
| **总数** | 总请求数量 | 次 |
| **失败** | 失败请求数量 | 次 |
| **成功率** | (总数-失败)/总数×100% | % |
| **平均RT** | 平均响应时间 | ms |
| **P50** | 50%请求的响应时间 | ms |
| **P90** | 90%请求的响应时间 | ms |
| **P99** | 99%请求的响应时间 | ms |
| **QPS** | 每秒请求数 | req/s |

**汇总行**显示所有请求的合计数据。
