## 🎯 输出时间线

测试运行过程中的输出顺序：

```

    ↓
每 5秒: 实时统计（请求数、成功率、QPS等）
    ↓
每 10秒: Metrics 数据表格（10个请求的详细指标）
    ↓
... 循环 ...
    ↓
测试结束: 最终统计 + 最终 Metrics 表格
```

## 💡 使用提示

### 1. 观察聚合器运行

由于聚合器默认**每15秒**执行一次聚合，所以：
- 测试开始后约**15-20秒**才会有第一次 Metrics 数据
- 如果显示"暂无 Metrics 数据（可能还未聚合）"，这是正常的

### 2. 理解数据延迟

```
发送 Mark → Recorder 记录 → 聚合器采集（15秒） → TSDB 存储
                                    ↑
                            脚本查询到的数据
```

所以查询到的数据会有最多15秒的延迟。

### 3. 配置聚合器时间间隔

如果想更快看到数据，可以调整聚合器的 `tickInterval`：

```go
// 在 provider 中修改
markAggregator := aggregator.NewMarkAggregator(
    ctx,
    5*time.Second,  // 改为5秒一次聚合
    tsdb,
    // ...
)
```

## 🎉 优势

1. **实时反馈**：无需等到测试结束就能看到数据
2. **清晰直观**：表格格式一目了然
