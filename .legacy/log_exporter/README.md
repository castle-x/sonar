### Log Exporter - 压测生态日志监控工具

#### 产品简介

Log Exporter 是一款专为压测场景设计的日志采集与监控工具，能够实时从游戏服务器日志中提取关键性能指标，并推送到监控平台，帮助测试团队及时发现性能问题。

#### 核心功能

##### 🔍 1. 智能日志发现
- **进程关联监控**：自动从进程命令行参数中发现日志文件路径，动态跟踪进程的日志输出
- **文件/目录监控**：支持直接指定日志文件路径或使用通配符监控多个日志文件
- **进程标签注入**：自动将进程 PID、启动参数等信息注入到监控标签中，实现进程与日志数据的关联

##### 📊 2. 灵活的指标提取
- **正则匹配**：通过正则表达式从日志内容中提取关键指标值
- **时间戳解析**：支持自定义时间戳格式，精确记录指标产生时间
- **采样密度控制**：可配置数据采样间隔，避免数据过于密集

##### 🚀 3. 实时数据推送
- **PushGateway 集成**：将采集到的指标推送到 PushGateway，接入完整监控体系
- **批量上报**：支持缓冲和批量上报，提高推送效率
- **全局标签**：支持配置集群、区域等全局标签，方便数据分类和查询

##### ⚡ 4. 高性能设计
- **实时监听**：基于文件系统事件监听，实时响应日志变化
- **内存友好**：只保留必要的指标数据，避免内存占用过高
- **优雅退出**：支持信号处理，确保进程正常退出时数据完整性

#### 典型应用场景

##### 游戏服务器压测监控
监控游戏服务器在压测过程中的关键性能指标：
- **帧率监控**：实时采集服务器平均帧率、低帧率事件等
- **玩家负载**：跟踪在线玩家数、服务器负载状态
- **异常检测**：及时发现性能告警和异常日志

##### 多进程服务监控
自动发现和监控多个服务进程的日志：
- 支持通过命令行参数过滤目标进程
- 自动为每个进程生成独立的监控指标
- 动态检测进程启动和退出，自动调整监控列表

##### 压测数据采集
在压测执行过程中实时采集关键数据：
- 配置灵活的采样密度，平衡数据完整性和系统开销
- 支持多维度标签，方便按场景、区域、集群等维度分析
- 与监控平台深度集成，实现自动告警和可视化展示

#### 快速开始

##### 配置文件示例
```yaml
log_config:
  # 从进程命令行参数获取日志路径
  - name: "GameServerMonitor"
    enabled: true
    rules:
      - name: "GameServer"
        cmdlines: ["--config", "!debug"]
        log_path_pattern: "-ABSLOG=(.+\\.log)"
    
    metrics:
      - name: "avg_fps"
        pattern: "AverageFps:(\\d+)"
        value: "$1"
        density: 15  # 每15秒采样一次

push_gateway:
  app_id: "pressure_test"
  enabled: true
  host: "http://pushgateway:8082"
  report_interval: 10
  labels:
    cluster: "test-cluster"
    region: "beijing"
```

##### 启动服务
```bash
./log_exporter -c config.yaml
```

#### 监控指标示例

Log Exporter 可以从日志中提取各类性能指标：

| 指标类型 | 示例 | 说明 |
|---------|------|------|
| 服务器帧率 | avg_fps, slow_fps | 游戏服务器性能核心指标 |
| 在线玩家 | online_player | 服务器负载情况 |
| 错误计数 | error_count | 异常日志统计 |
| 响应时间 | response_time | 请求处理延迟 |
