### Node Process Exporter - 压测生态监控系统

#### 产品概述

Node Process Exporter 是一款轻量级的分布式监控数据采集器，专为压测生态环境设计。它能够实时采集节点和进程级别的性能指标，并将数据推送至 PushGateway，为压测分析提供全方位的监控数据支撑。

#### 核心特性

##### 1. 📊 双层监控体系

###### 节点级监控 (Node Exporter)
- **系统资源监控**：全面采集节点级别的系统性能指标
  - 💻 CPU 使用率和负载
  - 🧠 内存使用情况（总量、已用、可用）
  - 🌐 网络流量统计（入站/出站）

###### 进程级监控 (Process Exporter)
- **精细化进程监控**：针对特定进程采集详细的运行指标
  - ⚡ 进程级 CPU 使用率
  - 💾 进程内存占用（RSS、VMS）
  - 📡 进程网络 I/O 统计
- **动态进程发现**：自动发现和监控符合规则的进程
- **智能标签提取**：从进程命令行参数中提取业务标签

##### 2. 🎯 灵活的进程匹配机制

支持多种进程识别方式：
- **PID 直接匹配**：通过进程 ID 精确监控
- **命令行参数过滤**：支持正向和反向匹配规则
  - ✅ 正向匹配：`"--config"` 匹配包含该参数的进程
  - ❌ 反向匹配：`"!seed"` 排除包含该关键词的进程

##### 3. 🏷️ 强大的标签提取能力

从进程命令行中自动提取标识信息，支持两种提取模式：

###### Split 分割提取
按指定分隔符分割命令行，提取指定位置的参数值：
```yaml
type: "split"
sep: " "
labels:
  ds_type: $1  # 提取第1个参数作为 ds_type 标签
```

###### Regex 正则提取
使用正则表达式匹配并提取特定模式的值：
```yaml
type: "regex"
pattern: "--id=(\\w+)"
labels:
  id: $1  # 提取 --id 参数的值
```

##### 4. 🚀 可靠的数据推送机制

- **批量推送**：支持缓冲区聚合，减少网络开销
- **可配置推送间隔**：根据压测需求灵活调整上报频率
- **超时控制**：避免网络异常导致的阻塞
- **全局标签**：支持为所有指标添加集群、区域等全局标识
- **本地日志输出**：可选择性打印指标日志便于调试

##### 5. ⚙️ 高性能采集引擎

- **定时采集**：支持秒级采集间隔配置
- **并发处理**：异步采集和推送，互不阻塞
- **动态刷新**：可配置进程列表动态刷新间隔
- **优雅关闭**：支持信号处理，确保数据完整性

#### 应用场景

##### 压测场景监控
- 实时监控压测机器和压测进程的资源消耗
- 通过标签识别不同压测任务的性能表现
- 为压测结果分析提供系统层面的数据支撑

##### 多实例服务监控
- 动态发现和监控同一服务的多个实例
- 自动提取实例 ID、配置类型等标识信息
- 支持细粒度的实例级性能对比

##### 分布式系统观测
- 统一采集多节点的系统和进程指标
- 通过全局标签区分不同集群和区域
- 集中推送到 PushGateway 进行汇总分析

#### 技术特点

- **语言实现**：基于 Go 语言开发，性能高效
- **轻量部署**：单一二进制文件，无外部依赖
- **配置驱动**：支持 YAML/JSON 配置文件
- **命令行友好**：基于 Cobra 框架，易于集成和使用
- **云原生适配**：设计理念符合 Prometheus 生态标准

#### 快速开始

##### 配置示例
```yaml
# 采集间隔
step: 3

# 推送网关配置
push_gateway:
  app_id: "my_stress_test"
  enabled: true
  host: "http://pushgateway:8082"
  report_interval: 10
  labels:
    cluster: "test-cluster"
    region: "beijing"

# 进程监控
process_exporter:
  enabled: true
  dynamic_interval: 15  # 15秒刷新进程列表
  rules:
    - name: "TestServer"
      cmdlines:
        - "--config"
        - "!seed"
      extracts:
        - type: "regex"
          pattern: "--id=(\\w+)"
          labels:
            server_id: $1

# 节点监控
node_exporter:
  enabled: true
```

##### 启动命令
```bash
./node_process_exporter -c config.yaml
```

#### 总结

Node Process Exporter 作为压测生态的数据采集基础设施，通过灵活的配置和强大的标签提取能力，为压测分析提供了细粒度、多维度的监控数据。它与 PushGateway、Prometheus、Grafana 等组件无缝集成，构建起完整的压测监控闭环，助力压测效率提升和问题定位。
